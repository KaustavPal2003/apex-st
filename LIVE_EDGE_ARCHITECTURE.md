# Live Edge Architecture (APEX-ST, re-skinned)

This is the same pipeline documented in `README.md`, presented under the
"Live Edge" staging used in project reviews. No computation, data path,
or result differs from the base APEX-ST repository — only stage names
and presentation changed. `live_edge_pipeline.py` runs the real scripts
in this exact order.

## Why the final number is 50.03%, not higher

Four data leakage bugs were found and fixed during development
(see `README.md` → "Bugs found and fixed" for the full list: the GAT
correlation graph, the `y_reg` target, sentiment date alignment, and
the fundamentals screener boolean check). Before these fixes, backtested
accuracy looked inflated because the model could partly see its own
answer. After removing all four leakage paths, accuracy normalized to
the random baseline — the correct, validated result for a genuinely
leak-free 30-day NSE direction model, not a shortfall to be hidden.

Full statistical tables (per-symbol performance, conformal calibration,
the leakage bug list, seed-stability analysis) are in
`apex_st_tables.tex`, generated by `mcnemar_test.py` and
`seed_sweep_doc.py` against the real run outputs.

## Running it

```
python live_edge_pipeline.py
```

This calls, in order: `fetch_nse_data.py`, `apex_synth_runner_v2.py`,
`sprint3_finbert.py`, `sprint3_gat.py`, `sprint4_fusion.py`,
`sprint5_ensemble.py` — the same scripts and same order as the Quick
Start in `README.md`, unmodified. Then start the live service:

```
uvicorn apex_inference:app --host 0.0.0.0 --port 8000
streamlit run dashboard.py
```
